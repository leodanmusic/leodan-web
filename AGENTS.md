# Reglas de trabajo para Codex en LEODAN

Estas reglas aplican a cualquier cambio dentro del proyecto LEODAN.

## Flujo de trabajo

1. Antes de modificar codigo, revisar la estructura del proyecto.
2. No cambiar links oficiales sin confirmacion.
3. No romper la landing principal.
4. Cada cambio debe explicar:
   - Que archivos se tocaron.
   - Que problema se resolvio.
   - Como probarlo.

## Marca y contenido

1. Mantener una estetica oscura, verde, elegante y minimalista.
2. Todo texto debe sonar como marca artistica profesional, no como plantilla generica.
3. Priorizar narrativa, autenticidad, energia y vision internacional.
4. Evitar textos genericos de DJ.
5. La URL principal para Instagram debe ser `https://leodan.net`.

## Frontend

1. Priorizar mobile-first porque la web se usa desde Instagram.
2. Crear componentes reutilizables para:
   - Lanzamientos.
   - Demos.
   - Press kit.
   - Sellos.
3. Mantener consistencia visual con la marca: oscuro, verde, elegante, minimalista y energetico.

## Paginas para sellos

Las paginas orientadas a sellos deben priorizar claridad comercial y decision rapida.

Deben incluir:

- Bio corta.
- Track destacado.
- Links privados o publicos.
- Press kit.
- Contacto.
- Storytelling breve.

El tono debe ser directo, artistico y profesional.

## Links oficiales

Los links oficiales no deben modificarse sin confirmacion explicita de LEODAN.

Referencia principal de marca y links: `docs/brand.md`.
